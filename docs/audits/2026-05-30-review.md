## 🟡 À surveiller / dette technique

### Sécurité (acceptable mais à surveiller)

- **`/api/changelog`** public : un user banni peut scraper en masse (limit 100 + cache atténue, donc pas critique)
- **`/api/rl-stats` + `/api/tm-stats`** complètement publics, ownership zéro, juste rate-limit IP — scraper-friendly pour les rangs RL de tous les Epic IDs publics
- **Cron `expire-invitations`** empile 4 passes (invitations + departures + Discord + Valorant) avec `maxDuration=60` — surveiller Sentry pour les passes incomplètes, découper en 2 crons quand plan Vercel le permettra
- **Dev seed** crée un doc dans `aedral_admins` (`app/api/dev/seed/route.ts:425`) — bien gaté `NODE_ENV !== 'development'`, mais doubler avec `VERCEL_ENV === undefined` (= machine locale uniquement) serait plus safe
- **Champs `metadata` audit logs** non sanitizés (peut faire exploser doc Firestore 1 MiB si endpoint forward du body brut)
- **`/api/profile/[uid]/rank-report`** : vérifier rate limit par couple (reporterUid, targetUid) anti-harassement

### UX / Polish

- **PlayerItem Switch custom** réimplémenté au lieu d'utiliser le design system (`app/community/players/page.tsx:786-805`) → extraire en `components/ui/Switch.tsx`
- **Loader2 brut partout** : `components/ui/Skeleton.tsx` n'est utilisé qu'**1 seule fois** dans tout le code malgré le design system. Listings (players, structures, replays, todos) → spinner centré au lieu de skeletons shimmer. Gain perçu énorme sur "premium".
- **Capitalization mixte sur tags** : certains UPPERCASE (OBLIGATOIRE, MIN. 1, ADMIN), d'autres Title Case (Recrute, Fait), d'autres lowercase (Stats, Parsing). Le `.tag` impose `text-transform: uppercase` donc c'est cosmétique côté code, mais à harmoniser au niveau source.
- **MonthView mobile `+N`** en 9px (`components/calendar/MonthView.tsx:217`) — justifiable contexte ultra-dense mais 11px tiendrait probablement
